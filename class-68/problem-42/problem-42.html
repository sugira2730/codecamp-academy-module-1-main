Programming Challenge

ATM Withdrawal Simulation using do...while

Problem Overview

In this challenge you will simulate a simple ATM withdrawal system. The objective is to practice algorithmic thinking and understand how the do...while loop works in JavaScript.

The ATM starts with an initial balance and allows the user to withdraw money repeatedly until they decide to stop or the balance reaches zero. Because the operation must run at least once, the solution must use a do...while loop.

Scenario

  • The ATM starts with an initial balance of $500.
  • The system asks the user how much money they want to withdraw.
  • If the withdrawal amount exceeds the balance, the system must display an insufficient funds message.
  • If the withdrawal is valid, the system updates and shows the remaining balance.
  • The user can continue making withdrawals until they choose to stop.
  • The program ends automatically if the balance becomes zero.

Your Tasks

1. Analyze the Problem

Identify the following elements:

  • Inputs
  • Outputs
  • Conditions
  • Loop termination condition

2. Design the Solution

Create a flowchart that represents the algorithm of the ATM system. Your flowchart should include:

  • Start and End nodes
  • User input
  • Decision conditions
  • Balance updates
  • The loop structure

3. Implement the Solution

Write a JavaScript program that:

  • Uses a do...while loop
  • Requests withdrawal amounts using prompt()
  • Displays messages using alert()
  • Validates the withdrawal amount
  • Updates and displays the remaining balance

Constraints

  • The loop must use do...while.
  • The balance cannot go below zero.
  • The program must execute at least once.
