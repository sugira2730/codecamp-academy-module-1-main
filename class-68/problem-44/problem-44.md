# Programming Challenge: Password Verification (Using `do...while`)

## Problem Overview

In this challenge, you will create a simple **password verification program**. 
The program will repeatedly ask the user to enter a password until the correct 
password is entered.

This exercise helps you practice the **`do...while` loop**, since the program 
must ask for the password **at least once** before checking if it is correct.

---

## Learning Objectives

By completing this exercise, you will practice:

- Understanding how the **`do...while` loop** works
- Designing a simple algorithm
- Comparing values in JavaScript
- Handling user input with `prompt()`
- Displaying messages with `alert()`

---

## Scenario

You are building a simple login verification system with the following behavior:

- The system has a **secret password** stored in the program.
- The program asks the user to **enter the password**.
- If the password is **incorrect**, the program asks the user to try again.
- The process repeats until the **correct password** is entered.
- When the correct password is entered, the program displays a **success message**.

---

## Your Tasks

### Analyze the Problem

Identify:

- Inputs
- Outputs
- Variables required
- Loop termination condition

---

### Design the Algorithm

Create a **flowchart** representing the logic of the program.

Your flowchart should include:

- Start and End nodes
- User input
- Password verification
- Loop structure

---

### Implement the Solution

Write a JavaScript program that:

- Uses a **`do...while` loop**
- Uses `prompt()` to ask the user for the password
- Compares the entered password with the correct password
- Uses `alert()` to display messages
- Stops only when the correct password is entered

---

## Constraints

- You **must use a `do...while` loop**
- The program must **execute at least once**
- The loop stops **only when the password is correct**

---

Enter the password: 1111

Enter the password: admin123
