JavaScript Conditionals

Professional guide to decision making in JavaScript

Introduction

Conditionals allow programs to make decisions. A program can execute different code depending on whether a condition is true or false.

In software engineering, conditionals are fundamental for building logic, validation systems, APIs, authentication flows, and user interfaces.

The if Statement

The if statement executes a block of code only if a condition evaluates to true. 


                if (condition) {
                // code executes if condition is true
                }

Example


                const age = 20;

                if (age >= 18) {
                    console.log("User is an adult");
                }

if / else

The else block runs when the condition is false. 


                const age = 16;

                if (age >= 18) {
                    console.log("Adult");
                } else {
                    console.log("Minor");
                }

else if

The else if statement allows evaluating multiple conditions. 


                const score = 85;

                if (score >= 90) {
                    console.log("Excellent");
                } else if (score >= 70) {
                    console.log("Good");
                } else {
                    console.log("Needs improvement");
                }

Comparison Operators

Operator Description Example
=== Strict equality a === b
!== Not equal a !== b
> Greater than a > b
< Less than a < b
>= Greater or equal a >= b
<= Less or equal a <= b

Logical Operators

AND ( && )

The AND operator requires both conditions to be true. 


                const age = 25;
                const hasLicense = true;

                if (age >= 18 && hasLicense) {
                    console.log("User can drive");
                }

OR ( || )

The OR operator requires at least one condition to be true. 


                const isAdmin = false;
                const isModerator = true;

                if (isAdmin || isModerator) {
                    console.log("User has access");
                }

NOT ( ! )

The NOT operator reverses a boolean value. 


                const loggedIn = false;

                if (!loggedIn) {
                    console.log("Please log in");
                }

Nested Conditionals

Sometimes conditionals must exist inside other conditionals. 


                const age = 20;
                const hasLicense = true;

                if (age >= 18) {

                    if (hasLicense) {
                        console.log("User can drive");
                    }

                }

Real World Example

A typical backend validation example. 


                const salary = 2500;
                const creditScore = 720;

                if (salary >= 2000 && creditScore >= 700) {
                    console.log("Loan Approved");
                } else {
                    console.log("Loan Rejected");
                }

Best Practices

  • Always use strict comparison (===)
  • Keep conditions simple
  • Avoid deeply nested conditionals
  • Use meaningful variable names
  • Extract complex logic into functions
